# 双模型编排方案概览

> 最后更新:2026-07-04

## 摘要

本方案用 Claude Pro 订阅的 Opus 承担"制定计划 / 写 workflow 脚本"这类高价值、低频次的工作,把"反复执行 / 多轮工具调用 / 长时间迭代"这类烧额度的事转移给自建 GPU 服务器上的 MiniMax M3,从而不消耗(或极少消耗)Anthropic 的 5 小时额度。方案分八步,第五步(工具调用测试)是判定方案可行性的关键节点。截至 2026-07-04,方案已完成落地并通过端到端验证。

## 一、目标与动机

- 用 Opus 做需求拆解、架构设计、写执行计划——高价值、低频。
- 把重复执行、多轮工具调用、长时间迭代的任务委派给自建的 MiniMax M3——不占 Anthropic 额度。

## 二、整体架构

```
主 session(Claude Pro, 手动切到 Opus)
  └─ 只负责:理解需求 → 拆解任务 → 写出一份可执行的 workflow 脚本/计划
       ↓
claude-m3(独立的 CLI 脚本,指向 M3,不占 Anthropic 额度)
  └─ 负责:按脚本反复执行 → 读文件、改代码、跑测试、修 bug、提交
```
