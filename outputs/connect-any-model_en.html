1. What this is, and why it's worth doing

If you use Claude Code (or a similar AI coding/work assistant), you've probably noticed: your usage quota is finite, and it runs out faster the more you use it. This document teaches you one thing — hand off the "repetitive legwork" to a cheaper, essentially unlimited AI model, and reserve your most capable, most expensive model for the parts that actually require judgment.

An analogy: Imagine you've hired a brilliant, expensive senior architect. They're best at making decisions, designing the plan, setting direction — things only they can do well. But if you have them fix a typo, re-run a test, or reformat ten files, you're wasting their time and your money. The smart move: let the architect make the calls, and hand the manual labor to a fast, inexpensive crew. This document walks you through setting up exactly that "crew."
Your strongest model (does the thinking)
Figures out what needs to happen, breaks it into clear steps
↓ hands off the clearly-defined steps
The newly connected model (does the legwork)
Repeatedly executes: reads/writes files, runs tests, fixes errors

Once set up, you'll have a new command. Keep using Claude Code exactly as before — just switch to this new command for the repetitive work, and your precious quota gets saved for the moments that actually need real judgment.

2. What you need before you start

You need two things:

  1. Claude Code, already installed If you haven't installed it yet, complete the official installation steps first and confirm that typing claude in your terminal opens it.
  2. Access to a model service that's "Anthropic-protocol compatible", plus the three pieces of information listed below, obtained from that provider. "Anthropic-protocol compatible" means the provider's API speaks the same "language" as Claude Code, so no extra translation layer is needed. Many providers explicitly advertise this as "Anthropic API compatible" or "Claude API compatible" in their docs — that's the phrase to look for.
What you needWhat it is (in plain terms)
Base URL Think of it as a "headquarters address" — where on the internet the service you're connecting to actually lives. It's typically a web address starting with https://, and your provider's documentation will state it explicitly.
API Key
(token)		 Think of it as your "membership card PIN" — proof that this is your account. Never show it to anyone else, and never paste it anywhere public (a chat group, a public code repository, etc.).
Model Name Think of it as "the exact dish name you order off the menu" — it tells the system which of the provider's models to use. It must match your provider's documentation exactly, including capitalization and any suffix (some providers use different name strings for a "standard" vs. an "extended-context" variant of the same underlying model — get this wrong and you may silently end up using a different variant than you intended).

If you don't have these three things yet, check your chosen provider's website — usually under an "API," "Developers," or "Getting Started" section.

Point your AI assistant at this HTML file (for example, just say: "Open this document and follow the 'Exact steps for an AI Agent' section to set this up for me"). It will ask you for the three pieces of information above, then handle everything else automatically.

4. Exact steps for an AI Agent

If you're an AI agent reading this, someone wants your help connecting Claude Code to another model. Follow the steps below in order. Anywhere you see a placeholder like {{like this}}, substitute the real value before running the command.

🔒
Security note: the API Key is sensitive. When confirming it back to the user, show only the first and last 4 characters (e.g. sk-a1***********wxyz). Never print the full key into logs, chat transcripts, or any file that might get committed to a repository.
  1. Step 0 · Environment check Confirm the claude command is installed. If not, stop and tell the user to install Claude Code first. 
    bashcommand -v claude >/dev/null 2>&1 || echo "claude command not found — install Claude Code first"
  2. Step 1 · Ask the user these three questions (verbatim)
    • "What's the Base URL? (Your provider's Anthropic-protocol-compatible endpoint, usually starts with https://)"
    • "What's the API Key? (The token your provider issued you)"
    • "What's the Model Name? (Must match your provider's documentation exactly, including case and any suffix)"
  3. Step 2 · Derive a short alias Turn model_name into a short, filesystem-safe alias: lowercase, replace every character that isn't a-z0-9 with a hyphen, collapse repeated hyphens, trim leading/trailing hyphens. 
    bashraw="{{MODEL_NAME}}"
alias=$(echo "$raw" | tr '[:upper:]' '[:lower:]' | sed -E 's/[^a-z0-9]+/-/g; s/^-+|-+$//g')
    If the result is empty or shorter than 2 characters (common when the model name is non-ASCII or symbol-heavy), ask a follow-up question: "Could you give this configuration a short nickname? e.g. minimax, kimi, zhipu."
  4. Step 3 · Check for a name collision Avoid silently overwriting an existing configuration. 
    bashtest -f "$HOME/.claude/settings-{{ALIAS}}.json" && echo "A config with this alias already exists — ask the user whether to overwrite or pick a different alias"
  5. Step 4 · Create the settings file Create the directory, then write ~/.claude/settings-{{ALIAS}}.json: 
    bashmkdir -p "$HOME/.claude"
    File contents (substitute the {{...}} placeholders): 
    json{
  "env": {
    "ANTHROPIC_BASE_URL": "{{BASE_URL}}",
    "ANTHROPIC_AUTH_TOKEN": "{{API_KEY}}",
    "ANTHROPIC_MODEL": "{{MODEL_NAME}}",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "{{MODEL_NAME}}",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "{{MODEL_NAME}}",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "{{MODEL_NAME}}",
    "API_TIMEOUT_MS": "1200000"
  }
}
    Why both ANTHROPIC_MODEL and the three ANTHROPIC_DEFAULT_*_MODEL fields: different Claude Code versions read slightly different fields, so setting all four maximizes compatibility — this combination has been tested and doesn't conflict. API_TIMEOUT_MS is set much higher than the default because third-party model endpoints can have a noticeably slow first response (one real test hit nearly three minutes).
  6. Step 5 · Create the executable wrapper Do not use a shell alias for this. AI coding assistants typically run commands in a non-interactive, non-login shell that never sources .zshrc / .bashrc, so aliases silently won't exist. It has to be a real executable file. 
    bashmkdir -p "$HOME/bin"
    File contents, written to ~/bin/claude-{{ALIAS}}: 
    bash#!/bin/bash
exec claude --settings "$HOME/.claude/settings-{{ALIAS}}.json" "$@"
    Make it executable: 
    bashchmod +x "$HOME/bin/claude-{{ALIAS}}"
  7. Step 6 · Confirm ~/bin is on PATH 
    bashecho "$PATH" | tr ':' '\n' | grep -qx "$HOME/bin" && echo OK || echo MISSING
    If it prints MISSING: check $SHELL to determine zsh vs. bash, append the following line to the corresponding ~/.zshrc or ~/.bashrc, and tell the user to restart their terminal or run source manually: 
    bashexport PATH="$HOME/bin:$PATH"
  8. Step 7 · Verify with test 1: text dialogue 
    bashclaude-{{ALIAS}} -p "introduce yourself in one sentence" --output-format json
    Check the returned JSON: "is_error" should be false and "result" should be non-empty. If PATH hasn't refreshed yet in this shell, use the full path ~/bin/claude-{{ALIAS}} instead.
  9. Step 8 · Verify with test 2: tool calling (the critical step) Run this in a scratch directory, and always include --allowedTools — without it, tool calls will hang waiting for a permission prompt that non-interactive mode can never answer. 
    bashmkdir -p /tmp/model-bridge-test && cd /tmp/model-bridge-test
claude-{{ALIAS}} -p "create a file named hello.txt in the current directory with the content hello" \
  --allowedTools "Write" --output-format json
    Don't just trust the model's self-reported success — independently check that the file actually exists with the right content, and confirm "permission_denials" in the JSON is an empty array.
  10. Step 9 · Report back to the user If steps 7 and 8 both pass, tell the user in plain language that setup succeeded, that they can now run claude-{{ALIAS}} "whatever they want done" from any folder, and remind them that any future automated (non-interactive) use needs --allowedTools or it will hang.

If a test fails, check in this order

SymptomCheck first
Immediate connection errorIs the Base URL correct and reachable? (try curl against it directly)
401 / 403 / unauthorizedWas the API Key copied completely, with no stray whitespace or line breaks?
Model not foundDoes the Model Name match the provider's docs exactly — case and suffix included?
Tool calls hang / nothing happensIs --allowedTools missing from the command?
Response is very slowRetry once — a slow first call to a third-party endpoint is a known, benign occurrence, not necessarily a misconfiguration

5. Frequently asked questions

Is this safe? Could my API key leak?

The settings file lives on your own machine, under ~/.claude/, and is never automatically uploaded anywhere. The one thing to watch for yourself: don't paste this file or its contents into a chat, or commit it to a public code repository. If a key does leak, go to your provider's dashboard, revoke it, and issue a new one.

Will this new command conflict with my existing claude command?

No. claude stays exactly as it was. The new command (e.g. claude-minimax) is entirely separate — the two don't interfere, and you can switch between them freely.

Why is it sometimes slow?

Response times vary a lot between third-party model providers, especially on the very first call (which can occasionally take a minute or two). That doesn't necessarily mean something's misconfigured — try again, or on a different network.

Can I connect more than one model provider?

Yes. Repeat the steps in Section 4 for each one, choosing a different alias each time (e.g. claude-minimax, claude-kimi) — each setup is independent and they won't interfere with each other.

I'd rather not have an AI agent do this automatically — can I do it by hand?

Absolutely. Every step in Section 4 is fundamentally "make a folder," "write a file," or "change a permission" — simple enough that even without a technical background, you can type the commands into a terminal yourself, one line at a time.

6. Glossary

API
A set of agreed-upon rules that let two pieces of software talk to each other. You don't need to know the details — just that it's the interface two programs use to communicate.
Base URL
The "street address," on the internet, of the service you're connecting to — usually a web address.
API Key (token)
A string of characters that proves "this is your account," functioning like a password. Never share it.
Environment variable
A configuration value a program reads on startup. The settings file in this guide is, at its core, a pre-written batch of these.
Permission prompt
The confirmation dialog an AI assistant shows before performing an action (like editing a file), asking "is this OK?" In unattended/automated scenarios, you must explicitly declare upfront which actions are allowed, or the program will sit stuck waiting for a click that never comes.
Anthropic-protocol compatible
Describes a model provider whose API is shaped exactly like Anthropic's (Claude's maker), so Claude Code can talk to it directly with no extra "translation layer" needed.
